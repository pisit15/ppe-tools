================================================================================
PROJECT OVERVIEW
================================================================================

A production-ready Personal Protective Equipment (PPE) Inventory Management 
System built with Next.js 14, TypeScript, Supabase, and Tailwind CSS.

Migrated from Coda to web application with:
- Modern UI/UX following eashe.org design system
- Multi-company support with company_id isolation
- Real-time inventory tracking
- Employee history tracking
- Comprehensive reporting

================================================================================
TECH STACK
================================================================================

Frontend:
  - Next.js 14 (App Router)
  - TypeScript (strict mode)
  - React 19
  - Tailwind CSS 4
  - Lucide React icons

Backend:
  - Supabase PostgreSQL
  - API routes (REST endpoints)
  - Row-level security (configurable)

Dev Tools:
  - ESLint
  - TypeScript compiler
  - Tailwind CSS PostCSS

PROJECT STRUCTURE
================================================================================

Key Directories:
  supabase/migrations/          Database schema + migrations
  src/app/                      Next.js pages and routes
  src/app/api/ppe/              REST API endpoints
  src/app/ppe/                  Main application pages
  src/components/               Reusable React components
  src/lib/                      Utilities and types

Key Files:
  supabase/migrations/001_ppe_tables.sql
    - 4 main tables (products, transactions, employees)
    - 1 computed view (stock summary)
    - Indexes and RLS policies

  src/lib/supabase.ts           Supabase client + helpers
  src/lib/types.ts              TypeScript interfaces (15 types)
  src/lib/constants.ts          App constants (PPE types, units, colors)

  src/app/page.tsx              Home/landing page
  src/app/ppe/page.tsx          Dashboard (stats & alerts)
  src/app/ppe/inventory/        Product CRUD
  src/app/ppe/stock-in/         Receive/return form
  src/app/ppe/stock-out/        Issue/borrow form
  src/app/ppe/history/          Transaction history
  src/app/ppe/employees/        Employee management
  src/app/ppe/reports/          Analytics and summaries

================================================================================
DATABASE SCHEMA
================================================================================

Tables:
  ppe_products              Product catalog
  ppe_transactions          Stock movements (in/out/return/borrow)
  ppe_employees             Employee directory
  ppe_stock_summary (VIEW)  Computed current stock levels

Features:
  - company_id based multi-tenancy
  - Full audit trail (created_at timestamps)
  - Stock calculation view
  - Indexed for performance

================================================================================
API ENDPOINTS
================================================================================

All endpoints support company_id query parameter for multi-tenancy.

Products:
  GET  /api/ppe/products?company_id=XXX
  POST /api/ppe/products

Transactions:
  GET  /api/ppe/transactions?company_id=XXX&[filters]
  POST /api/ppe/transactions

Stock Summary:
  GET  /api/ppe/stock?company_id=XXX
  GET  /api/ppe/stock?company_id=XXX&low_stock=true

Employees:
  GET  /api/ppe/employees?company_id=XXX
  POST /api/ppe/employees

ENVIRONMENT SETUP
================================================================================

Required Environment Variables:
  NEXT_PUBLIC_SUPABASE_URL       Supabase project URL
  NEXT_PUBLIC_SUPABASE_ANON_KEY  Public anon key
  SUPABASE_SERVICE_ROLE_KEY      Service role key (keep secret)

Optional:
  NEXT_PUBLIC_APP_NAME           App display name
  NEXT_PUBLIC_DOMAIN             Deployment domain

Setup Steps:
  1. Copy .env.local.example to .env.local
  2. Add Supabase credentials from project Settings > API
  3. Run migration SQL in Supabase
  4. Start development server
